# 水軍與長江／黃河移動（設計備忘）

> 狀態：**規劃中**，尚未接入 `MovementRules` / 尋路。  
> 陸地：地形 tier **0 / 8 / 12 / 16**，日行動池 = 兵種 **`mobility`**（見前文討論）。

---

## 為什麼要這樣設計：三國志14 的問題與本專案解法

### 14 的結構性問題

| 現象 | 後果 |
|------|------|
| 部隊**一下水就變鬥艦**（或頂級水軍形態） | 水陸兩用過強，無專職分工 |
| **到處上岸**、繞後包圍 | 防守方無「險」可守，江防線形同虛設 |
| 東吲地理優勢難體現 | 長江天險變成「誰都能無腦渡江」 |

結果：**東吲無險可守**——不是江不寬，是規則讓大軍可以任意水格↔岸格切換。

### 本專案對策（核心）

```text
大軍渡江（蒙衝及以上）→ 必須「港灣 → 港灣」
走舸（navy.small）     → 可任意靠岸，但怕激流、運量小
```

由此產生 **14 缺少的戰役節奏**：

| 層級 | 玩法 |
|------|------|
| **戰略** | 要過江？先奪或壓制 **港灣**（襄陽、江陵、武昌、建业…），不能從任意河岸卸大軍 |
| **牽制** | 進攻方必須 **派兵壓制港灣**、吸引／釘住對方 **主力**，否則渡江隊在港外被截 |
| **戰術破局** | **走舸** 在港灣戰場側翼 **靠岸**，做 **陸上突破**（小股、斷道、燒糧），不是用大船到處包圍 |

東吲「有險可守」= **港灣鏈 + 激流 + 寬江**；曹魏強渡 = **攻港 + 走舸偷岸** 的組合拳，而不是單一「鬥艦無限上岸」。

### 與兵種線的對應

| 單位 | 角色（呼應上文） |
|------|------------------|
| 走舸 | 側翼登陸、偵查、接舷；**不能**代替大軍從港灣投送 |
| 蒙衝+ | 江面主力、過激流；**大軍**只在此系統下從港灣进出 |
| 樓船／鬥艦 | 火力與攻城平台，不是「一下水就變身無敵包圍」 |

**設計底線**：部隊**不會**因進入水格就自動變成鬥艦；水軍編制與陸軍編制分離，下水＝已帶水軍兵種且遵守上表規則。

| 項目 | 決策 |
|------|------|
| 戰略格尺度 | **10 km / 格** |
| **長江、黃河** | 地圖上畫得**很寬**（多格水域帶） |
| 玩法目的 | 江面是走廊（巡邏、南北增援）；**港灣**控制渡江與大船隊 |

## 水上移動：每格扣 4 行動力

| 項目 | 決策 |
|------|------|
| 消耗 | 水軍在**一般水格**上，**每進 1 格扣 4 點行動力**（不是「每天固定扣 4」） |
| 對比陸地 | tier-1 **8**、tier-2 **12**、tier-3 **16** |

### 渡江節奏（為何不拖大行軍）

寬江常 **約 10～20 格**（100～200 km 帶狀水域），橫渡約 **5～10 格**直線水路：

| 橫渡水路格數 | 僅水路消耗（×4） | 約天數（走舸 mobility 24） |
|--------------|------------------|----------------------------|
| 6 格 | 24 | **~1 天**（滿池剛好） |
| 12 格 | 48 | **~2 天** |
| 15～20 格 | 60～80 | **~3～4 天** |

若主力是 **蒙衝 mobility 20**：同樣 12 格約 **3 天**。  
加上**激流繞行、水戰、港灣進出、陆军集结**，整體 **橫渡寬江約 3～5 個遊戲日** → 比純陸軍長途奔襲慢，但**不會卡死節奏**，也留給東吳反應時間。

## 激流（Rapids）

| 項目 | 決策 |
|------|------|
| 地形 | 水上格可標 **激流** |
| 走舸 `navy.small` | **不可過**激流 |
| 蒙衝 `navy.medium` 及以上 | 需 **研發科技** 後方可過激流 |
| 流向 | **不模擬**順逆流 |

走舸：**最快**，但被激流限制走位；蒙衝是科技 + 兵種分水嶺。

## 港灣與靠岸（提高港灣價值）

### 核心規則

| 行為 | 走舸 `navy.small` | 蒙衝及以上／走舸以外渡江 |
|------|-------------------|---------------------------|
| **在江上行走** | 水格每格 **-4** | 同左 |
| **渡河（軍隊橫江）** | **不必**經港灣；可從任意適合的岸格接舷 | **必須**從 **港灣** 登船出發，到達對岸也須在 **港灣** 下船 |
| **靠岸 / 登陸** | **任意靠岸**（鄰接岸上格即可） | **不需**在港灣「停泊」才能在江上活動；但 **陸軍搭大船渡江** 仍只能 **港灣进出** |

### 設計意圖（一句話）

- **走舸** = 輕便：偷渡、偵查、小隊登陸、接舷——**繞過港灣**（對應「走舸旁邊上岸做陸上突破」）。  
- **蒙衝以上 / 大船隊** = 主力與物資：**港灣是咽喉**；進攻方須 **派兵壓制港灣、牽制主力**，大軍才能港→港渡江。  
- 「**其他也不需在港灣靠岸**」= 蒙衝、樓船、鬥艦 **平時在江上不用每回合回港灣泊船**；與走舸差在 **渡江登陸權** 與 **激流**，不是差在「能不能在江上待著」。

### 典型攻防腳本（備忘）

**北岸攻東吲（如曹魏）**

1. 主力東向壓 **武昌／江陵港灣**，逼東吲水軍決戰或分兵。  
2. 蒙衝以上船隊 **僅能** 在己方奪港後，港→港 投送大軍（3～5 日水路節奏）。  
3. **走舸** 自港戰側翼任意靠岸：斷補給、占高地、配合主力合擊——**不能**单靠走舸把全城围死。

**東吲防守**

1. 守 **港灣 + 激流** 節點，遲滯大船隊。  
2. 走舸反制走舸，蒙衝對蒙衝；避免「任一格上岸即崩」。  
3. 江寬 + 水格 -4：南北增援仍比純陸路快，體現 **天險在我**。

### 陸軍（備忘）

- 陸軍**不能**當水軍走開放水格。  
- 陸軍渡江方式：  
  - **走舸接舷**（任意岸，小規模）；或  
  - **港灣 + 蒙衝及以上**（大軍團橫江）。

### 地圖要素（待做）

| 地形 / 建築 | 用途 |
|-------------|------|
| **港灣** | 大船隊渡江、卸載、貿易、造船（可擴） |
| **渡口**（可選） | 與港灣不同：輕渡／棧道，不取代「大船必須過港」 |
| **任意岸格** | 僅 **走舸** 可讓部隊登陸 |

## 水軍兵種線

```text
走舸 [navy.small] → 蒙衝 [navy.medium] → 樓船 [navy.large] → 鬥艦 [navy.final]
```

| 階段 | 規劃角色 |
|------|----------|
| 走舸 | 最快（水格 -4）；**任意靠岸**；**不過激流**；小規模渡江 |
| 蒙衝 | 激流（+科技）；江面主力；**大軍渡江必經港灣** |
| 樓船 / 鬥艦 | 火力、攻城；同蒙衝港灣規則；江上不需每日回港 |

數值見 `Assets/StreamingAssets/chinese/unit.properties`。
